# Silver Annotation Policy

## Purpose

Silver annotation is used to expand training data while the project waits for a
manually resolved gold test set.

## Rules

- Silver labels may be generated by deterministic rules, lexicons, or LLM
  assistance.
- Silver labels may be used for training and error discovery.
- Silver labels must not be described as human annotation.
- Silver labels must not be used to compute inter-annotator agreement.
- Final performance claims must use `data/manual_gold/gold_resolved.conll`.

## Current Implementation

`src/build_silver_dataset.py` generates `data/silver/train.conll`,
`data/silver/val.conll`, and `data/silver/test.conll` with the current
rule-and-lexicon annotator.

The active training config uses:

```yaml
training:
  train_data_source: silver
```
